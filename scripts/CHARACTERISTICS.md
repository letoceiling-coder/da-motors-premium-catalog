# Анализ получаемых характеристик автомобилей

## ✅ Что мы получаем из API донора

### Основная информация
- ✅ **Марка** (brand) - извлекается из названия
- ✅ **Модель** (model) - извлекается из названия
- ✅ **Комплектация** (trim) - извлекается из названия (базовая)
- ✅ **Год выпуска** (year) - парсится из описания или названия
- ✅ **Цена** (price) - извлекается из API, автоматически исправляется если некорректная
- ✅ **Статус** (status) - в наличии / в пути / под заказ

### Технические характеристики
- ✅ **Тип топлива** (fuelType) - парсится из описания (Бензин/Дизель/Гибрид/Электрический)
- ✅ **Объем двигателя** (engineVolume) - парсится из описания (например, "2.0 л")
- ✅ **Мощность** (power) - парсится из описания (например, "237 л.с.")
- ✅ **Тип коробки передач** (transmission) - парсится из описания (автоматическая/механическая/робот)
- ✅ **Привод** (drivetrain) - парсится из описания (полный/передний/задний)
- ✅ **Тип кузова** (bodyType) - определяется по названию модели (sedan/suv/coupe/hatchback/convertible/wagon)

### Визуальная информация
- ✅ **Фотографии** (photos) - извлекаются из галереи товара
- ✅ **Цвет** (color) - парсится из описания или названия (белый, черный, серый, и т.д.)

### Дополнительно
- ✅ **Пробег** (mileage) - парсится из описания, если указан (по умолчанию 0 для новых)
- ✅ **VIN** (vin) - извлекается из данных продукта, если есть, иначе генерируется

## ⚠️ Что мы НЕ получаем (ограничения API)

### Детальные характеристики
- ❌ **Подвеска** (suspension) - пустой объект, не предоставляется API
- ❌ **Безопасность** (safety) - пустой массив, не предоставляется API
- ❌ **Комфорт** (comfort) - пустой массив, не предоставляется API
- ❌ **Мультимедиа** (multimedia) - пустой массив, не предоставляется API
- ❌ **Дополнительное оборудование** (additional) - пустой массив, не предоставляется API

### Причины ограничений

1. **API Tilda не предоставляет детальные характеристики** - API возвращает только базовую информацию о товаре (название, цена, описание, фото)

2. **Описание ограничено** - В поле `descr` обычно содержится только краткая информация вида "2.0 л | 237 л.с. | Дизель | 2024"

3. **Нет структурированных данных** - API не возвращает детальные спецификации в структурированном виде

## 🔍 Как улучшить получение характеристик

### Вариант 1: Парсинг страницы товара
Можно добавить парсинг индивидуальных страниц товаров для получения детальной информации:
- Открыть страницу каждого автомобиля
- Извлечь детальные характеристики из HTML
- Заполнить поля suspension, safety, comfort, multimedia, additional

### Вариант 2: Использование дополнительных API endpoints
Если у Tilda есть дополнительные endpoints для получения детальной информации о товаре, можно их использовать.

### Вариант 3: Ручное дополнение
Детальные характеристики можно добавлять вручную для премиум-автомобилей.

## 📊 Текущее состояние парсинга

### Успешно парсится:
- Марка и модель: ✅
- Цена: ✅ (с автоисправлением)
- Год: ✅
- Двигатель (объем, мощность, тип топлива): ✅
- Коробка передач: ✅ (из описания)
- Привод: ✅ (из описания)
- Тип кузова: ✅ (определяется по названию)
- Цвет: ✅ (из описания/названия)
- Пробег: ✅ (если указан)
- Фото: ✅
- Статус: ✅

### Не парсится (нет в API):
- Детальные характеристики подвески: ❌
- Системы безопасности: ❌
- Опции комфорта: ❌
- Мультимедиа системы: ❌
- Дополнительное оборудование: ❌

## 💡 Рекомендации

1. **Для базового каталога** - текущего парсинга достаточно, так как получаем все основные характеристики
2. **Для детального каталога** - можно добавить парсинг страниц товаров для получения полной информации
3. **Для премиум-сегмента** - детальные характеристики можно добавлять вручную
